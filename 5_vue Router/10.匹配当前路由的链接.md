# 匹配当前路由的链接

我们一般在导航栏里用 router-link 组件来匹配当前路由的链接，但我们想做一些视觉的呈现, 告诉用户当前页面的位置。

有两个 css 类可以帮助我们实现这个效果：

- router-link-active: activeClass

- router-link-exact-active: exactActiveClass

## 匹配规则

1. 它与当前路径匹配相同的路由记录

2. 它的 params 与当前路径的 params 相同

3. 如果使用嵌套路由, 任何指向祖先路由的链接会被匹配, 只要相关的 parmas 匹配

4. 其他路由属性, 如 query, 不会影响匹配

5. 使用 alias 也被认为是匹配的

6. 如果有 redirect, 不会匹配

## 精确匹配当前路由的链接

一般是嵌套路由使用的

因为匹配规则第三个, 所以哪个 path 更精准, 就会添加对应的类, router-link-exact-active

## 配置类名

```html
<RouterLink active-class="active" exact-active-class="exact-active"
  >tab1</RouterLink
>
```

```js
const router = createRouter({
  linkActiveClass: "active",
  linkExactActiveClass: "exact-active",
});
```
